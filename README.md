## 🏗️ Architecture Principles

```
┌─────────────────────────────────────────────────────┐
│              Hexagonal Architecture                  │
│                                                      │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐      │
│   │  REST    │    │  Domain  │    │   DB     │      │
│   │ Adapter  │───▶│  Core    │◀───│ Adapter  │      │
│   └──────────┘    │  (DDD)   │    └──────────┘      │
│                   └──────────┘                       │
│   ┌──────────┐         ▲      ┌──────────┐           │
│   │   SQS    │         │      │  PSP     │           │
│   │ Adapter  │─────────┘      │ Adapter  │           │
│   └──────────┘                └──────────┘           │
└─────────────────────────────────────────────────────┘
```

- **Hexagonal Architecture** — strict separation of domain, application and infrastructure
- **Domain-Driven Design** — Aggregates, Value Objects, Domain Events
- **Event-Driven** — Outbox Pattern, SQS, async processing
- **TDD** — test-first, mutation testing, >85% coverage target
- **SOLID & Clean Code** — enforced via code reviews and static analysis
